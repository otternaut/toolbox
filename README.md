# 🧰 toolbox

**A collection of portable, tool-agnostic AI agent skills.**

Reusable instruction sets that teach coding agents — Claude Code, Codex, Cursor,
and others — how to perform specific tasks well.

Each skill is a self-contained directory of Markdown. A single install script
symlinks every skill into the global skills directory of each supported tool — so
a skill authored once is available **everywhere**, and edits take effect
immediately with no reinstall required. ✨

## ✅ Requirements

- 🐚 **bash** and **coreutils** — preinstalled on macOS and Linux.
- 🛠️ At least one supported AI tool that reads skills from its config directory:

  | Tool | Skills directory |
  | --- | --- |
  | Claude Code | `~/.claude/skills` |
  | Codex | `~/.codex/skills` |
  | Cursor | `~/.cursor/skills` |

No language runtime, package manager, or build step is needed. 🎉

## 🚀 Installation

Clone the repository and run the installer:

```bash
git clone https://github.com/otternaut/toolbox.git
cd toolbox
./scripts/install
```

Want to see what would happen first? Preview it with a dry run:

```bash
./scripts/install --dry-run
```

Because skills are **symlinked, not copied**, any edit you make in this repo is
picked up by every tool immediately — no need to re-run the installer after
changing a skill. 🔗

The installer is **idempotent and non-destructive**:

- ♻️ Re-running refreshes stale symlinks and leaves correct ones untouched.
- 🛡️ Real files or directories that are not symlinks are never overwritten —
  they're skipped with a warning.
- 📁 Missing target directories are created automatically.
- 🪝 It points this repo's git at `.githooks` (via `core.hooksPath`) so the
  README skills table stays in sync automatically — see
  [Keeping docs in sync](#-keeping-docs-in-sync).

Run `./scripts/install --help` for all options.

## 💡 Using a skill

Once installed, invoke a skill from your agent by name or just by describing the
task. For example, with `analyze-code`:

```
/analyze-code https://github.com/acme/widgets/pull/42
```

…or simply:

```
review my changes before I open a PR
```

The agent matches your request against each skill's `description` and loads the
right one for you. 🎯

## 🔄 Keeping docs in sync

The [Available skills](#-available-skills) table is **generated**, not
hand-maintained. [`scripts/update-readme`](scripts/update-readme) reads the
`description` from each `skills/<name>/SKILL.md` and rewrites the table between
the `<!-- BEGIN/END SKILLS TABLE -->` markers.

After you run `./scripts/install`, a **pre-commit git hook**
([`.githooks/pre-commit`](.githooks/pre-commit)) runs this generator on every
commit and re-stages `README.md` if the table changed — so adding, removing, or
re-describing a skill updates the README automatically. You never edit the table
by hand. ✍️

You can also run it manually:

```bash
./scripts/update-readme          # rewrite the table now
./scripts/update-readme --check  # exit non-zero if the table is stale (for CI)
```
